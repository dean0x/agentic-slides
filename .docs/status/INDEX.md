## Usage
- **Starting a session?** - Use `/catch-up` command to restore context
- **Ending a session?** - Use `/devlog` to document progress
- **Need full context?** - Read the full status documents
- **Quick reference?** - Use compact versions

## Document Types

### Full Status (`.docs/status/YYYY-MM-DD_HHMM.md`)
Comprehensive capture including:
- Session context and decisions
- Todo list state for continuity
- Project state analysis
- Architecture documentation
- Next steps and context

### Compact Status (`.docs/status/compact/YYYY-MM-DD_HHMM.md`)
Quick reference including:
- Focus summary
- Key accomplishments
- Critical decisions
- Next actions
- Essential context

## Session Continuity

To restore a previous session's context:
1. Read the latest full status document
2. Recreate the todo list using the preserved JSON
3. Review "Next Steps" section for priorities
